# 统一时间戳和用户字段命名规范变更提案

## 背景

目前在多个微服务中，实体类的时间戳和用户字段命名不一致：
- 某些实体使用 `modifiedUser` 字段
- 某些实体使用 `updateUser` 字段
- 时间戳和用户字段的命名缺乏一致性

这种不一致性导致了以下问题：
1. 代码风格不统一
2. 增加开发人员理解成本
3. 容易在实现业务逻辑时出现错误

## 变更目标

统一所有微服务中实体类的时间戳和用户字段命名，具体如下：
- 创建时间字段：统一为 `createTime`
- 修改时间字段：统一为 `updateTime`
- 创建人字段：统一为 `createUser`
- 修改人字段：统一为 `updateUser`

## 影响范围

涉及以下微服务：
1. user-service
2. shopping-service
3. volunteerActivity-service

## 实施方案

1. 修改所有实体类中的字段命名
2. 更新相应的Mapper接口和XML文件
3. 修改Service层实现类中的字段引用
4. 确保数据库表结构兼容性（如需要则提供迁移脚本）

## 风险评估

低风险：主要为字段名称变更，不影响业务逻辑。
