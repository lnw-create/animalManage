# 扩展UserContext功能提案

## 背景

目前的UserContext工具类只能获取用户名，但在某些业务场景下需要同时获取用户ID。例如，在一些需要记录操作日志或关联用户数据的场景中，用户ID比用户名更加可靠和高效。

## 问题描述

现有的UserContext.java类只提供了获取用户名的方法，没有提供获取用户ID的方法，导致在某些需要用户ID的业务场景中无法直接获取。

## 解决方案

扩展UserContext.java类，增加对用户ID的支持，包括：

1. 在ThreadLocal中增加用户ID的存储
2. 提供设置和获取用户ID的方法
3. 修改相关的拦截器以支持从请求头获取用户ID
4. 修改网关过滤器以传递用户ID信息

## 预期收益

1. 便于在微服务中获取用户ID
2. 提高数据关联操作的效率
3. 减少根据用户名查询用户ID的额外开销
